# Пропорит — приложение-напоминалка

Полный исходный код Android-приложения на Kotlin + Jetpack Compose.
Внутри уже лежит файл `.github/workflows/build-apk.yml`, который сам соберёт
`.apk` в облаке GitHub — тебе не нужно ничего устанавливать на компьютер.

## Как получить .apk (пошагово)

1. **Зайди на github.com** и зарегистрируйся (если ещё нет аккаунта) — бесплатно.
2. Нажми **New repository** (зелёная кнопка справа сверху). Назови как хочешь,
   например `proporit-app`. Сделай его **Public** или **Private** — не важно.
   Не добавляй README/gitignore при создании — оставь всё пустым.
3. Открой созданный репозиторий → **Add file → Upload files**.
4. Перетащи в окно **всё содержимое** этой папки (`proporit-app`), включая
   скрытую папку `.github` — если сайт не даёт загрузить папки целиком,
   загрузи сначала обычные файлы и папки (`app`, `gradle`, `build.gradle.kts`,
   `settings.gradle.kts`, `gradle.properties`, `gradlew`, `gradlew.bat`), а
   `.github/workflows/build-apk.yml` — отдельным шагом (создай файл через
   **Add file → Create new file**, вставь путь `.github/workflows/build-apk.yml`
   и вставь содержимое).
5. Нажми **Commit changes**.
6. Перейди во вкладку **Actions** вверху репозитория. Сборка запустится
   автоматически (займёт 2-4 минуты). Если не запустилась — нажми на workflow
   слева и **Run workflow**.
7. Когда сборка позеленеет (✅), открой её и внизу страницы в разделе
   **Artifacts** будет файл `proporit-debug-apk` — скачай и распакуй (это zip,
   внутри — `app-debug.apk`).
8. Перекинь `app-debug.apk` на телефон (через USB, Telegram, почту — как удобно)
   и открой файл — Android спросит разрешение установить из неизвестного
   источника, разреши (это нормально для приложений не из Google Play).

## После установки — обязательно сделай на телефоне

- Разреши уведомления, когда приложение спросит при первом запуске.
- Зайди в **Настройки приложения → Настройки** внутри самого «Пропорита» —
  если там будет жёлтая карточка про «точные напоминания», нажми
  **Открыть настройки** и включи «Будильники и напоминания» — без этого
  Android может присылать напоминания с опозданием.
- Проверь, что для приложения не включена агрессивная экономия батареи
  (у Xiaomi/Huawei/Samsung это отдельная настройка) — иначе система может
  «убивать» приложение в фоне и напоминания перестанут приходить.

## Что внутри приложения

- **Главная** — кольцо-таймер (сколько дней осталось), кнопка
  «Поменял пропорит», серия подряд успешных замен (золотой значок).
- **Календарь** — история замен и плановая дата следующей.
- **Настройки** — интервал замены (по умолчанию 3 дня), частота повторных
  напоминаний (по умолчанию раз в час), звук/вибрация, тихие часы.

Логика полностью «самовосстанавливающаяся»: следующая дата замены всегда
считается как *время последнего нажатия кнопки + интервал*, и пересчитывается
заново при каждом запуске и после перезагрузки телефона — так что если
приложение упадёт или телефон перезагрузится, график не собьётся.

## Если сборка в Actions упадёт с ошибкой

Открой лог упавшего шага (обычно `Build debug APK`) и пришли мне текст ошибки —
я поправлю код. Версии библиотек подобраны стабильные, но Google иногда
снимает с публикации совсем старые версии инструментов, так что стоит быть
готовым к одной-двум мелким правкам после первой реальной сборки.
